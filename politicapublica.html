1

Quem somos

O Gabriel é um assistente pessoal inteligente desenvolvido e operado de forma individual. O aplicativo está disponível em https://atlasgabriel.netlify.app e funciona como uma Progressive Web App (PWA) instalável em dispositivos móveis.

Esta Política de Privacidade descreve como tratamos os dados dos usuários do Gabriel, incluindo os dados obtidos através da integração com serviços do Google (Gmail e Google Agenda), em conformidade com a Lei Geral de Proteção de Dados (LGPD — Lei nº 13.709/2018) e com as Políticas de Dados do Google.

2

Dados que coletamos

O Gabriel coleta apenas os dados estritamente necessários para o funcionamento do assistente pessoal. Todos os dados são armazenados localmente no dispositivo do usuário (IndexedDB do navegador), exceto quando há integração ativa com serviços do Google.

devices
Armazenamento 100% local: Conversas, memórias, tarefas, eventos, notas e gastos financeiros ficam salvos exclusivamente no seu dispositivo. Nenhum dado pessoal é enviado para servidores do Gabriel.
Tipo de dado Onde fica Finalidade
Nome e e-mail Dispositivo local Identificação e personalização
Histórico de conversas Dispositivo local Contexto para o assistente
Memórias extraídas Dispositivo local Personalização das respostas
Eventos e agenda Dispositivo local Lembretes e notificações
Registros financeiros Dispositivo local Controle pessoal de gastos
Chave API Groq Dispositivo local (criptografada) Acesso ao modelo de IA
Tokens Google OAuth Dispositivo local Acesso a Gmail e Agenda
3

Uso dos dados do Google (Gmail e Agenda)

security
Conformidade com a Política de Dados do Usuário do Google: O uso de dados obtidos pelas APIs do Google segue estritamente os Serviços de API do Google — Política de Dados do Usuário, incluindo os requisitos de Uso Limitado.

Escopos solicitados e sua finalidade:

Escopo Google Para que é usado
gmail.readonly Leitura de e-mails para exibição e resumo pelo assistente
gmail.send Envio de e-mails a pedido do usuário via chat
gmail.modify Marcar e-mails como lidos quando o usuário solicitar
calendar.readonly Exibição de eventos do Google Agenda no dashboard
calendar.events Criação de eventos no Google Agenda a pedido do usuário
userinfo.email / profile Identificação da conta Google conectada

O Gabriel não irá, em nenhuma circunstância:

  • Usar dados do Gmail ou Google Agenda para publicidade ou criação de perfil para fins comerciais
  • Transferir dados do Google para terceiros, exceto quando necessário para fornecer o serviço ao próprio usuário
  • Usar dados do Google para desenvolver, melhorar ou treinar modelos de inteligência artificial ou machine learning
  • Armazenar conteúdo de e-mails em servidores externos — os dados ficam apenas no dispositivo do usuário
  • Vender dados de usuários ou dados obtidos via APIs do Google
  • Permitir que humanos leiam os dados do Google sem consentimento explícito do usuário, exceto em caso de requisito legal

O acesso aos dados do Google é feito diretamente do dispositivo do usuário para as APIs do Google, sem passar por servidores intermediários do Gabriel, exceto a Netlify Function responsável pela troca inicial do código OAuth por tokens de acesso — que não armazena nenhum dado.

4

Como usamos seus dados

Os dados coletados são utilizados exclusivamente para:

  • Personalizar as respostas do assistente com base no histórico e preferências do usuário
  • Criar e gerenciar eventos, tarefas, notas e registros financeiros conforme solicitado
  • Exibir informações relevantes no painel (dashboard) como clima, agenda e tarefas
  • Enviar notificações push de lembretes quando solicitado pelo usuário
  • Integrar com Gmail e Google Agenda exclusivamente a pedido do usuário

As mensagens enviadas ao assistente são processadas pela API Groq utilizando a chave fornecida pelo próprio usuário. O Gabriel não tem acesso a essas mensagens — elas são enviadas diretamente do dispositivo do usuário para os servidores da Groq.

5

Armazenamento e segurança

Todos os dados do Gabriel são armazenados localmente no dispositivo do usuário usando o IndexedDB do navegador. Isso significa que:

  • Nenhum dado pessoal é enviado ou armazenado em servidores do Gabriel
  • Os dados existem apenas enquanto o usuário mantiver o aplicativo instalado
  • A senha do usuário é armazenada como hash SHA-256 — nunca em texto puro
  • Tokens de acesso do Google são armazenados no localStorage do dispositivo
  • O usuário pode apagar todos os dados a qualquer momento limpando os dados do site no navegador
warning_amber
Atenção: Como os dados ficam no dispositivo, limpar os dados do navegador ou desinstalar o app resultará na perda permanente de todos os dados locais do Gabriel. Recomendamos exportar informações importantes regularmente.
6

Compartilhamento de dados

O Gabriel não compartilha, vende, aluga ou transfere dados pessoais dos usuários para terceiros, exceto nas seguintes situações estritamente necessárias:

  • API Groq: As mensagens de chat são enviadas à API Groq para processamento pela IA. Consulte a Política de Privacidade da Groq.
  • APIs do Google: Quando o usuário conecta sua conta Google, os dados são enviados às APIs do Google conforme os escopos autorizados. Consulte a Política de Privacidade do Google.
  • Netlify: O app está hospedado na Netlify. O acesso ao app passa pela infraestrutura da Netlify. Consulte a Política de Privacidade da Netlify.
  • Exigência legal: Podemos divulgar informações se exigido por lei ou ordem judicial.
7

Seus direitos (LGPD)

De acordo com a Lei Geral de Proteção de Dados (LGPD), você tem os seguintes direitos:

  • Acesso: Saber quais dados temos sobre você
  • Correção: Corrigir dados incompletos ou incorretos
  • Exclusão: Solicitar a exclusão dos seus dados
  • Portabilidade: Receber seus dados em formato estruturado
  • Revogação: Revogar o consentimento a qualquer momento
  • Oposição: Opor-se ao tratamento de dados

Como os dados ficam no seu próprio dispositivo, você tem controle total. Para revogar o acesso do Google, acesse myaccount.google.com/permissions e remova o acesso do Gabriel.

8

Retenção de dados

Os dados são retidos pelo tempo em que o usuário mantiver o aplicativo ativo no dispositivo. Não há retenção em servidores externos. O usuário pode excluir todos os dados a qualquer momento:

  • Limpando os dados do site no navegador (Configurações → Privacidade → Dados do site)
  • Desinstalando o aplicativo PWA do dispositivo
  • Usando a função de logout que limpa os dados de sessão
9

Alterações nesta política

Esta Política de Privacidade pode ser atualizada periodicamente. Quando isso ocorrer, a data de "Última atualização" no topo desta página será alterada. Recomendamos revisar esta página regularmente.

Mudanças significativas serão comunicadas diretamente no aplicativo antes de entrarem em vigor.
